# 第4周 Day2：SQL条件筛选进阶

## 1. 今日学习内容

本次学习了以下SQL语法：

- IS NULL：筛选空值
- IS NOT NULL：筛选非空值
- IN：筛选多个指定值
- NOT IN：排除多个指定值
- BETWEEN：筛选数字或日期范围
- LIKE：模糊匹配文本
- OR：多个条件满足其一
- NOT LIKE：排除符合文本特征的数据
- CASE WHEN：根据条件生成分类字段

## 2. NULL的理解

SQL中的空值使用NULL表示，读取到Python中可能显示为None或NaN。

判断空值不能使用：

    字段 = NULL

应该使用：

    字段 IS NULL
    字段 IS NOT NULL

NULL不会自动被IN或NOT IN选中。

如果业务需要在排除指定值的同时保留空值，需要单独增加：

    OR 字段 IS NULL

## 3. 数据范围意识

一条订单可能销售额有效，但日期或城市缺失。

因此：

- 销售额有效的订单可以参与整体销售分析。
- 日期缺失的订单不能参与月份分析。
- 城市缺失的订单不能归入明确城市，但不能因此从所有分析中删除。
- 取消订单可以保留用于订单质量分析，但不参与正式销售指标。

## 4. BETWEEN范围筛选

BETWEEN可以用于筛选数字和日期，并且包含上下边界。

例如：

    net_sales BETWEEN 800 AND 1200

相当于：

    net_sales >= 800 AND net_sales <= 1200

日期筛选示例：

    order_date BETWEEN '2026-02-01' AND '2026-02-28'

日期为空的订单不会被日期范围条件选中。

## 5. LIKE模糊匹配

LIKE用于按照文字特征筛选数据。

- LIKE '鼠标%'：以“鼠标”开头
- LIKE '%鼠标'：以“鼠标”结尾
- LIKE '%鼠标%'：任意位置包含“鼠标”
- NOT LIKE '%鼠标%'：名称中不包含“鼠标”

## 6. CASE WHEN分类

本次将有效订单按照销售额分类：

- 大于等于1200：高销售额
- 大于等于800且小于1200：中销售额
- 小于800：低销售额

分类结果：

- 高销售额：2条，销售额3317.85
- 中销售额：7条，销售额7469.00
- 低销售额：6条，销售额3447.30

三组订单数合计15条，销售额合计14234.15，与有效订单总体指标一致。

分类标准改变会影响各组的订单数、销售额和平均值，但不会改变原始订单总数和总销售额。

## 7. 独立练习

业务需求：

查询天猫和抖音渠道的有效销售订单，并将销售额大于等于1000的订单标记为重点订单。

查询结果：

- 有效订单：11条
- 重点订单：6条
- 普通订单：5条

由于样本数量较少，而且订单明细尚未按渠道汇总，不能仅凭该结果判断天猫或抖音哪个渠道表现更好。

## 8. 今日总结

SQL筛选必须根据业务问题明确数据范围。

空值不代表整条记录无用，应该判断缺失字段是否会影响当前分析。

CASE WHEN可以把连续指标转换为业务分类，但分类门槛需要依据实际业务标准确定。
