# 第4周 Day4：SQL综合业务分析

## 1. 今日目标

本次综合使用以下SQL知识：

- SELECT与字段别名
- WHERE明细筛选
- INNER JOIN多表关联
- GROUP BY多维度分组
- HAVING分组结果筛选
- ORDER BY指标排序
- COUNT、SUM和ROUND
- 模拟成本、模拟毛利和模拟毛利率
- CASE WHEN条件计数
- 数据覆盖率检查
- 分组指标与总体指标核对
- 业务结论和数据限制表达

## 2. 结构化复习

完整的数据分析流程为：

1. 确认数据表和字段
2. 检查数据质量
3. 明确有效数据范围
4. 确定分析维度和指标
5. 必要时关联其他表
6. 分组计算指标
7. 排序或筛选结果
8. 与整体指标进行一致性检查
9. 说明数据覆盖率和结论限制

WHERE用于分组前筛选订单明细。

HAVING用于分组后筛选统计结果。

## 5. 规模指标与效率指标

模拟毛利反映当前销售产生的毛利金额规模。

模拟毛利率反映每1元销售额能够留下多少模拟毛利。

本次结果中：

- 天猫电脑配件模拟毛利最高，为2214.45
- 京东电脑配件模拟毛利率最高，为51.32%

因此，模拟毛利最高的组合不一定模拟毛利率最高。

## 6. 高价值组合筛选

筛选标准：

- 有效订单数不少于2条
- 模拟毛利率不低于45%

符合条件的组合：

- 天猫电脑配件
- 京东电脑配件
- 天猫数码产品

抖音数码产品的模拟毛利率为31.41%，因此没有进入筛选结果。

筛选标准是本次练习设定的业务规则，不代表固定行业标准。

无法归入月份的指标为：

- 销售额：2294.30

因此，月份结论只能描述日期有效的12条订单。

## 10. 业务结论

当前样本中，天猫电脑配件组合的销售额和模拟毛利最高，京东电脑配件的模拟毛利率最高。

在日期有效的订单中，2026年3月销售额和模拟毛利最高，天猫是3月的主要贡献渠道。

但当前样本数量较少、日期覆盖率只有80%，并且标准成本为模拟数据，因此这些结果只能描述当前样本现象，不能证明长期经营能力或因果关系。
