# 第5周 Day4：SQL综合分析与结课验收

## 一、今日学习目标

本次练习综合使用 SQL 完成用户复购、购买间隔、Cohort 留存和用户价值分层分析。

主要知识包括：

- WHERE 筛选有效业务数据
- GROUP BY 分组统计
- COUNT、SUM、AVG 聚合计算
- CASE WHEN 业务分类
- CTE 分步组织查询
- LAG 获取上一笔记录
- RANK 生成用户价值排名
- SUM() OVER() 计算整体指标
- COUNT(DISTINCT) 去重统计用户
- 指标一致性验证

## 二、数据范围

本次使用 `user_orders` 用户订单表。

- 原始订单记录：17 条
- 发生有效购买的用户：8 人
- 有效订单总销售额：6470 元

有效订单筛选条件：

```sql
WHERE order_status = '已完成'
```

已取消订单不参与复购、留存和销售额指标计算。

## 三、用户复购指标

- 购买用户数：8 人
- 复购用户数：5 人
- 复购率：62.5%
- 跨月复购用户数：4 人
- 跨月复购率：50%
- 有效订单总销售额：6470 元

指标口径：

- 复购用户：有效订单数达到 2 笔及以上的用户。
- 跨月复购用户：在至少 2 个不同月份发生有效购买的用户。
- 复购率：复购用户数除以全部购买用户数。
- 跨月复购率：跨月复购用户数除以全部购买用户数。

同一个用户在同月购买两次，属于订单复购，但不属于跨月复购。

## 四、复购间隔分析

使用 LAG 获取用户上一笔有效购买日期：

```sql
LAG(order_date) OVER (
    PARTITION BY user_id
    ORDER BY order_date
)
```

使用 `julianday()` 计算两次购买之间的天数：

```sql
julianday(order_date) - julianday(previous_purchase_date)
```

结果：

- 复购行为数：7 次
- 复购用户数：5 人
- 复购间隔累计：235 天
- 平均复购间隔：33.57 天
- 最短复购间隔：12 天
- 最长复购间隔：54 天

33.57 天的统计单位是每次复购行为，不是每名用户。多次复购的用户会产生多条间隔记录。

## 五、Cohort 留存矩阵

`cohort_month` 表示用户第一次完成有效购买的月份：

- `month_number = 0`：首次购买月
- `month_number = 1`：首次购买后的第 1 个月
- `month_number = 2`：首次购买后的第 2 个月

| Cohort 月份 | Cohort 人数 | 第1月留存人数 | 第1月留存率 | 第2月留存人数 | 第2月留存率 |
|---|---:|---:|---:|---:|---:|
| 2026-01 | 3 | 1 | 33.33% | 2 | 66.67% |
| 2026-02 | 3 | 2 | 66.67% | NULL | NULL |
| 2026-03 | 2 | NULL | NULL | NULL | NULL |

必须区分：

- 0%：已经进入观察期，但没有用户留存。
- NULL：观察时间还没到，暂时无法计算。

1 月 Cohort 的第 2 月留存率高于第 1 月，不一定是计算错误。用户可能第 1 月没有购买，但第 2 月重新回来购买，这属于用户回流。

## 六、用户价值分层

分层规则：

- 核心用户：有效订单数不少于 2 笔、活跃月份数不少于 2 个月，并且总销售额不少于 1000 元。
- 复购用户：不满足核心用户条件，但有效订单数不少于 2 笔。
- 普通用户：其余发生过有效购买的用户。

分层结果：

- 核心用户：U001、U002、U006
- 复购用户：U004、U008
- 普通用户：U003、U005、U007

使用 RANK 按照用户总销售额排名：

```sql
RANK() OVER (
    ORDER BY total_net_sales DESC
)
```

## 七、用户分层汇总

| 用户分层 | 用户数 | 分层销售额 | 用户平均销售额 | 销售额占比 |
|---|---:|---:|---:|---:|
| 核心用户 | 3 | 3580 | 1193.33 | 55.33% |
| 复购用户 | 2 | 1890 | 945.00 | 29.21% |
| 普通用户 | 3 | 1000 | 333.33 | 15.46% |

在当前模拟数据中，核心用户虽然只有 3 人，但贡献了 55.33% 的有效销售额。

## 八、一致性验证

各用户分层销售额合计：

```text
3580 + 1890 + 1000 = 6470
```

各用户分层销售额占比合计：

```text
55.33% + 29.21% + 15.46% = 100%
```

分层销售额之和与有效订单总销售额 6470 元一致，本次分层统计通过一致性检查。


## 十一、数据限制

本次数据是用于学习的小规模模拟数据：

- 只有 8 名购买用户；
- 观察期只有 3 个月；
- Cohort 人数只有 2 至 3 人；
- 单个用户会明显影响比例；
- 尚未进入观察期的数据不能按 0% 处理。

因此，本报告中的数值只用于验证 SQL 分析方法，不能用于推断真实业务表现。
