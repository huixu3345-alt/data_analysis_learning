# 第2周 Day 1 数据质量检查报告

## 一、数据背景

本次练习使用 students_dirty.csv，模拟真实数据中常见的缺失值问题。

## 二、原始数据规模

- 原始数据行数：8
- 原始数据列数：8

## 三、缺失值检查结果

每列缺失值数量如下：

student_id     0
name           0
class_name     0
gender         0
math           1
english        1
python         1
enroll_date    1

## 四、发现的问题

1. 英语、数学、Python成绩中存在缺失值。
2. enroll_date 入学日期字段存在缺失值。
3. 如果直接删除所有缺失行，会导致样本数量减少。
4. 如果直接用平均值填充成绩，需要在报告中说明处理逻辑。

## 五、处理方法

本次练习采用以下处理方式：

1. 成绩类字段使用该字段平均值填充。
2. 日期字段 enroll_date 使用“未知日期”填充。
3. 填充后重新计算 total_score 和 avg_score。
4. 使用 groupby 按班级统计平均成绩。

## 六、初步结论

1. 缺失值会影响总分、平均分和班级统计结果。
2. 缺失值不能盲目删除，需要根据字段含义决定处理方式。
3. 数据清洗是分析前的必要步骤，否则后续结论可能不可靠。

## 七、补充分析结论

本次数据共 8 条学生记录，其中 math、english、python、enroll_date 各存在 1 个缺失值，单字段缺失比例均为 12.5%。

如果直接使用 dropna 删除所有包含缺失值的记录，数据会从 8 行减少到 4 行，样本损失达到 50%。因此，本次没有直接删除缺失记录，而是对成绩类字段使用均值填充，对入学日期字段使用“未知日期”填充。

填充缺失值后，重新计算了 total_score 和 avg_score，并按班级进行了分组统计。从结果看，B 班平均分最高，A 班整体表现较稳定，C 班平均分最低。但由于 C 班样本量只有 2 人，结论需要谨慎解读。

本次练习说明，数据清洗会直接影响后续统计结果。在进行分组统计、排序和指标计算前，需要先检查数据质量，并在分析报告中说明缺失值处理方法。
